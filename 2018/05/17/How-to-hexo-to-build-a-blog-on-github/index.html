How to use hexo to build a blog on github

为什么要放在Github Page？

  • 免费的资源，且社区强大，是程序员的聚集地。
  • 数据安全可靠，如果自己搭建服务器去存放个人博客，迁移与维护是一个问题，而且还需要去配置网络服务器对外暴露，比如Nginx或者Apache等容器。
  • 操作简便，只需要几个Git命令或者使用了插件之后，只需要一个部署命令即可完成部署操作
  • 由于选用Hexo来做个人博客，不需要后端服务支持，Github Page只支持静态HTML页面与一些CSS，JS，Img等文件，所以应用的场景是支持的
  • Github Page支持我们自定义 domain 可以将自己的域名与Github做链接

准备环境

  • 系统 macOS v10.13.4
  • node v6.11.2
  • npm 5.5.1
  • git 编辑器->Atom
  • 以上为本人搭建服务器的时候所处的环境

Push到Github上的步骤

  • 相信您已经有了Github账号啦，没有也不急，可以参照这个链接去注册 点击这里
  • 注册完之后，进入主页，新建一个项目 Create a new repository
  • 勾选所需的步骤，创建一个项目
  • 进入项目，找到项目的Settings选项 找到GitHub Pages中的 Source 选择分支 一般为master 这里有一点如果您想用自己的域名，可以选择 Custom domain 这个选项 填上您的域名，如何在您的域名商解析的时候选用 CNAME 解析 在此为放一个链接供参考域名设置

      • 进入选项后 右上角点击 New SSH key

      • 在命令行中执行 ssh -T git@github.com 如果正常返回您的账户名，说明您已经成功的免密链接到你的github账号啦，我的返回值是 Hi cjm61177! You've successfully authenticated, but GitHub does not provide shell access.
